timers

2019-04-15 • 标签: node.js

When writing JavaScript code, you might want to delay the execution of a function. Learn how to use setTimeout and setInterval to schedule functions in the future

当写js代码的时候，你可能需要延迟执行某个方法。现在来学习下如果用setTimeout和setInterval来定时执行。

setTimeout(callback, timeout)

setTimeout接受2个参数，第一个callback代表你将要延迟执行的方法，第二个参数代表你将要延迟的时间，以milliseconds为单位。

下面的句法定义了新的function，你可以直接调用这个方法，或者把它当成一个参数传递给setTimeout延时执行

setTimeout方法返回一个timer id，大部分情况下用不到它，但是可以存储它，以便你在不需要它定时执行的时候清除它。

Zero delay

如果把timeout指定为0，那么程序会尽可能快的执行callback，并且在当前函数执行之后。

This is especially useful to avoid blocking the CPU on intensive tasks and let other functions be executed while performing a heavy calculation, by queuing functions in the scheduler.

setInterval(callback, interval)

setInterval和setTimeout类似，不同的是setTimeout只执行一次结束了，而setInterval会以一定的间隔不停的执行callback。

上面这个方法会每2秒执行一次callback，除非你告诉它停止运行，通过setInterval返回的timer id。

Note: 可以在callback内部执行clearInterval方法，来停止定时函数的执行。

setImmediate

相当于setTimeout(callback, 0)，多用于node.js事件循环里
